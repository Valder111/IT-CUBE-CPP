## **Теория по Git. Основы**

### **Что такое Git?**
Git — это **система контроля версий**, которая помогает отслеживать изменения в коде, возвращаться к предыдущим версиям и работать в команде без конфликтов.

### **Что такое GitHub?**
GitHub — это онлайн-платформа, где можно хранить **репозитории** (проекты под управлением Git). Это как «облако» для вашего кода, доступное отовсюду.

### **Основные понятия**
- **Репозиторий (repo)** — папка проекта, за которой следит Git.
- **Коммит (commit)** — «снимок» состояния проекта. Каждый коммит сохраняет изменения и содержит описание.
- **Индекс (staging area)** — промежуточная зона: вы выбираете, какие файлы войдут в следующий коммит.
- **Remote** — ссылка на удалённый репозиторий (например, на GitHub).
- **.gitignore** — файл, в котором перечисляются файлы и папки, которые **не нужно отслеживать** (например, временные файлы Visual Studio: `*.suo`, `Debug/`, `*.user` и т.п.).

### **Основные команды**
| Команда | Назначение |
|--------|------------|
| `git init` | Инициализировать новый репозиторий |
| `git add <файл>` | Добавить файл в индекс |
| `git commit -m "сообщение"` | Сделать коммит с описанием |
| `git remote add origin <URL>` | Связать локальный репозиторий с удалённым |
| `git push -u origin main` | Отправить изменения на GitHub |
| `git clone <URL>` | Клонировать удалённый репозиторий (с GitHub) в локальное место |

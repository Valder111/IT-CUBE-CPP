# Передача аргументов в функции: по значению, по ссылке, по указателю

## Краткое напоминание 
- **Значение (`int x`)**  
  Создаётся копия переменной. Изменения внутри функции **не влияют** на оригинал.
- **Ссылка (`int& x`)**  
  "Псевдоним" оригинальной переменной. Изменения **затрагивают** оригинал.  
  → Не может быть `nullptr`, нельзя переопределить.
- **Указатель (`int* x`)**  
  Хранит адрес переменной. Требует разыменования (`*x`).  
  → Может быть `nullptr`, можно менять адрес.

---

## Способы передачи 

### 1. Передача **по значению**
```cpp
void increment(int a) {
    a++; // изменяется КОПИЯ
}
int x = 5;
increment(x);
cout << x; // 5 (не изменилось)
```
**Когда использовать:**  
- Маленькие данные (числа, bool).
- Когда нужно защитить оригинал от изменений.

---

### 2. Передача **по ссылке**
```cpp
void increment(int& a) {
    a++; // изменяется ОРИГИНАЛ
}
int x = 5;
increment(x);
cout << x; // 6
```
**Особенности:**  
- Нельзя передать литерал: `increment(10)` → ошибка.
- Для защиты от изменений: `void print(const int& a)`.

---

### 3. Передача **по указателю**
```cpp
void increment(int* a) {
    (*a)++; // изменяется ОРИГИНАЛ
}
int x = 5;
increment(&x);
cout << x; // 6
```
**Особенности:**  
- Можно передать `nullptr`: `increment(nullptr)`. Нужно проверять!
- Удобно для работы с массивами:
  ```cpp
  void printArray(int* arr, int size) {
      for (int i = 0; i < size; i++) cout << arr[i] << " ";
  }
  ```

## Сравнительная таблица
| Критерий          | По значению       | По ссылке (&)        | По указателю (*)     |
|--------------------|-------------------|-------------------|-------------------|
| Изменяет оригинал? | Нет               | Да                | Да                |
| Допустимо `nullptr`?   | Нет               | Нет               | Да                |
| Скорость           | Медленно для больших объектов | Быстро | Быстро |
